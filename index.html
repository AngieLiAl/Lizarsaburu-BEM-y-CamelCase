Metodologías de nombrado: BEM y camelCase

Esta página tiene como objetivo explicar y comparar dos metodologías de nombrado muy utilizadas en el desarrollo web: BEM y camelCase. Además, se aplican los conceptos básicos de HTML y CSS aprendidos en clase.

¿Qué es BEM?

BEM (Block Element Modifier) es una metodología de nombrado para clases en CSS que ayuda a escribir código más claro, estructurado y mantenible. Se basa en dividir la interfaz en bloques independientes que contienen elementos y pueden tener modificadores.

Ventajas de usar BEM

  1. Facilita la reutilización de componentes.
  2. Mejora la legibilidad del código.
  3. Hace más fácil el mantenimiento del CSS.
  4. Evita conflictos de nombres.
  5. Favorece el trabajo en equipo.

Ejemplos de clases con BEM

  • .tarjetaProducto
  • .tarjetaProducto__titulo
  • .tarjetaProducto__imagen
  • .tarjetaProducto--destacada
  • .boton--deshabilitado

¿Qué es camelCase?

camelCase es una convención de escritura donde la primera palabra comienza en minúscula y las siguientes empiezan con mayúscula, sin espacios ni guiones. Es muy utilizada para nombrar variables y funciones en JavaScript y clases en CSS.

¿Dónde se usa camelCase?

  1. Nombres de variables en JavaScript.
  2. Funciones en lenguajes de programación.
  3. Clases en frameworks JS.
  4. Nombres de archivos en proyectos web.
  5. Métodos en objetos JavaScript.

Ejemplos de camelCase

  • miVariable
  • obtenerDatosUsuario
  • formularioDeContacto
  • menuDesplegable
  • cambiarColorFondo

Comparación entre BEM y camelCase

Característica BEM camelCase
Uso principal Clases CSS Variables y funciones JS
Sintaxis bloque__elemento--modificador primeraLetraMinúscula
Claridad estructural Alta Media
Legibilidad Alta Alta
Ideal para Componentes HTML y CSS Lógica de programación
✅ Recomendación: BEM se recomienda para estructurar clases CSS en proyectos grandes y colaborativos. camelCase se usa más en la lógica del código, especialmente en JavaScript.
